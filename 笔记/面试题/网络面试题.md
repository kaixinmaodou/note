# 网络面试题

> 高频计算机网络面试题精选，涵盖 HTTP、TCP、缓存、跨域等核心知识点。

---

## 1. HTTP 和 HTTPS 的区别

| 对比 | HTTP | HTTPS |
| --- | --- | --- |
| 端口 | 80 | **443** |
| 加密 | 明文传输 | **SSL/TLS 加密** |
| 证书 | 不需要 | 需要 CA 证书 |
| 性能 | 快 | 略慢（多一次握手） |

---

## 2. HTTP 状态码

| 状态码 | 含义 |
| --- | --- |
| 200 | 成功 |
| 301 | 永久重定向 |
| 302 | 临时重定向 |
| 304 | 协商缓存命中，资源未修改 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
| 502 | 网关错误 |
| 503 | 服务不可用 |

---

## 3. GET 和 POST 的区别

| 对比 | GET | POST |
| --- | --- | --- |
| 参数位置 | URL 中 | 请求体中 |
| 参数长度 | 受 URL 长度限制（2KB左右）| 无限制 |
| 安全性 | 参数暴露在 URL 中 | 相对安全 |
| 缓存 | 会被浏览器缓存 | 不会 |
| 幂等性 | 幂等（多次相同结果） | 非幂等 |
| 后退/刷新 | 无影响 | 会重新提交 |
| 编码 | URL 编码 | 支持多种编码 |

> **本质上** GET 和 POST 都是 TCP 连接，在 HTTP 协议层面才有区别。上述区别大部分是**浏览器层面**的限制，而非协议限制。

## 4. TCP 三次握手 / 四次挥手

**三次握手（建立连接）：**
```
客户端 → SYN=1, seq=x           → 服务器    (SYN_SENT)
客户端 ← SYN=1, ACK=1, ack=x+1 ← 服务器    (SYN_RCVD)
客户端 → ACK=1, ack=y+1         → 服务器    (ESTABLISHED)
```

**四次挥手（断开连接）：**
```
客户端 → FIN → 服务器   (FIN_WAIT_1)
客户端 ← ACK ← 服务器   (CLOSE_WAIT)
客户端 ← FIN ← 服务器   (LAST_ACK)
客户端 → ACK → 服务器   (TIME_WAIT → 2MSL后CLOSED)
```

## 5. 强缓存和协商缓存

```
浏览器请求资源
    ↓
强缓存命中？（Cache-Control / Expires）
    ├─ 是 → 直接使用缓存（200 from cache）
    └─ 否 → 协商缓存
              ↓
    发请求到服务器（携带 If-Modified-Since / If-None-Match）
        ├─ 资源未变 → 304 Not Modified，使用缓存
        └─ 资源已变 → 200，返回新资源
```

| 类型 | 字段 | 优先级 |
| --- | --- | --- |
| 强缓存 | `Cache-Control: max-age` | 高（优先于 Expires） |
| 强缓存 | `Expires` | 低（绝对时间，有时差问题） |
| 协商缓存 | `ETag` / `If-None-Match` | 高（精确到内容） |
| 协商缓存 | `Last-Modified` / `If-Modified-Since` | 低（精确到秒） |

## 6. 跨域解决方案

1. **CORS**（最常用）：服务端设置 `Access-Control-Allow-Origin`
2. **代理**：开发环境 webpack/vite 代理，生产环境 Nginx 反向代理
3. **JSONP**：利用 `<script>` 不受同源策略限制（仅支持 GET）
4. **WebSocket**：不受同源策略限制
5. **postMessage**：跨窗口通信

---

## 7. 输入 URL 到页面展示的完整过程

1. **URL 解析**：浏览器解析 URL，判断是否有缓存
2. **DNS 解析**：域名 → IP 地址（递归查询）
3. **TCP 连接**：三次握手建立连接
4. **TLS 握手**：如果是 HTTPS，进行 TLS 握手
5. **发送 HTTP 请求**：构建请求报文发送给服务器
6. **服务器处理请求**：后端程序处理，返回响应
7. **浏览器解析渲染**：
   - 解析 HTML → DOM 树
   - 解析 CSS → CSSOM 树
   - 合并 → 渲染树
   - 布局（Layout）→ 绘制（Paint）→ 合成（Composite）
8. **TCP 断开**：四次挥手
