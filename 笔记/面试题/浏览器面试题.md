# 浏览器面试题

> 高频浏览器相关面试题精选，涵盖进程模型、渲染、存储、事件循环等核心知识点。

---

## 1. 浏览器进程模型

- **浏览器主进程**：界面管理、子进程管理
- **网络进程**：网络请求
- **渲染进程**：每个标签页一个，执行 HTML/CSS/JS
- **GPU 进程**：图层合成、3D 绘制
- **插件进程**：运行浏览器插件

---

## 2. 浏览器渲染流水线

```
解析HTML → 样式计算 → 布局 → 分层 → 绘制 → 分块 → 光栅化 → 画
```

---

## 3. 浏览器存储

| 方式 | 大小 | 过期 | 请求携带 |
| --- | --- | --- | --- |
| Cookie | 4KB | 可设过期时间 | **每次自动携带** |
| localStorage | 5MB | 永久 | 不携带 |
| sessionStorage | 5MB | 关闭标签页 | 不携带 |
| IndexedDB | 大 | 永久 | 不携带 |

---

## 4. Cookie / Session / Token

- **Cookie**：服务端设置，浏览器自动携带（存在 CSRF 风险）
- **Session**：存储在服务端，通过 Cookie 中的 SessionID 关联
- **Token (JWT)**：服务端签发，客户端存储，请求时主动带上（无状态、可跨域）

## 5. 事件循环（Event Loop）

```
宏任务 → 清空所有微任务 → 渲染（如果需要）→ 下一个宏任务 → ...
```

### 经典面试题

```js
console.log(1);

setTimeout(() => {
  console.log(2);
  Promise.resolve().then(() => console.log(3));
}, 0);

Promise.resolve().then(() => {
  console.log(4);
  setTimeout(() => console.log(5), 0);
});

console.log(6);

// 输出: 1 6 4 2 3 5
```

**分析：**

1. 同步代码：`1`、`6`
2. 微任务：`4`（Promise），执行过程中注册了 setTimeout(5)
3. 宏任务：`2`（setTimeout），执行过程中产生微任务 `3`
4. 微任务：`3`
5. 宏任务：`5`

## 6. 计时器能精确计时吗？

**不能**。原因：

1. 硬件层面没有原子钟，计时本身有偏差
2. 操作系统的计时函数有最小精度限制
3. W3C 标准：嵌套超过 5 层时，最短间隔为 4ms
4. **事件循环机制**：回调函数只能在主线程空闲时才能执行，如果主线程正忙，就必须等待

## 7. requestAnimationFrame

- 在浏览器**下一次重绘之前**执行
- 执行频率与屏幕刷新率一致（通常 60fps = 16.7ms/帧）
- 适合做动画，比 `setTimeout` 更精确流畅

```js
function animate() {
  element.style.transform = `translateX(${x}px)`;
  x += 1;
  if (x < 300) {
    requestAnimationFrame(animate);
  }
}
requestAnimationFrame(animate);
```
