# 工程化面试题

> 高频前端工程化面试题精选，涵盖模块化、构建工具、性能优化等核心知识点。

---

## 1. ES Module 和 CommonJS 的区别

| 对比 | ES Module | CommonJS |
| --- | --- | --- |
| 语法 | `import/export` | `require/module.exports` |
| 加载 | **静态分析**（编译时确定） | 动态加载（运行时） |
| 值 | **引用**（绑定） | **拷贝**（值） |
| 异步 | 支持 | 同步 |
| Tree-shaking | ✅ | ❌ |

---

## 2. Webpack 和 Vite 的区别

- **Webpack**：先打包再启动，项目越大越慢
- **Vite**：基于原生 ESM 按需编译，秒启动

---

## 3. Webpack Loader 和 Plugin 的区别

| 对比 | Loader | Plugin |
| --- | --- | --- |
| 作用 | **转换**文件（如 less → css → js） | **扩展**功能 |
| 运行时机 | 模块编译时 | 整个构建生命周期 |
| 使用方式 | `module.rules` | `plugins` 数组 |
| 本质 | 函数（接收源码，返回转换后代码） | 类（apply 方法监听钩子） |

---

## 4. Webpack 构建流程

```
1. 初始化参数：合并配置文件 + 命令行参数
2. 开始编译：创建 Compiler 对象，加载所有插件
3. 确定入口：根据 entry 找到所有入口文件
4. 编译模块：从入口出发，调用 loader 翻译模块，递归处理依赖
5. 完成编译：得到每个模块翻译后的内容和依赖关系图
6. 输出资源：根据依赖关系组装成 chunk，转换为文件
7. 输出完成：将文件写入 output 指定的目录
```

---

## 5. chunk、bundle、module 的区别

- **module**：源文件，一切皆模块
- **chunk**：打包过程中的代码块（由多个 module 组成）
- **bundle**：打包后的输出文件

---

## 6. css-loader 和 style-loader 的区别

- `css-loader`：解析 CSS 中的 `@import` 和 `url()`，将 CSS 转为 JS 模块
- `style-loader`：将 CSS 通过 `<style>` 标签注入到 DOM
- 生产环境用 `MiniCssExtractPlugin.loader` 替代（提取为独立 CSS 文件）

## 7. Source Map

| 选项 | 速度 | 适用 |
| --- | --- | --- |
| `eval-cheap-module-source-map` | 快 | **开发推荐** |
| `source-map` | 慢 | **生产推荐** |
| `hidden-source-map` | 慢 | 生产（上传错误监控） |
| `nosources-source-map` | 慢 | 生产（防源码泄露） |

## 8. HMR 热更新原理

```
1. 文件修改 → webpack 增量编译生成 hot-update.json + hot-update.js
2. webpack-dev-server 通过 WebSocket 通知浏览器
3. HMR Runtime 接收通知 → JSONP 请求 hot-update.js
4. 局部替换模块，不刷新页面
```

---

## 9. 如何优化构建速度？

1. 缩小搜索范围：resolve.alias、include/exclude
2. 缓存：babel-loader 的 cacheDirectory
3. 多进程：thread-loader
4. DLL：DllPlugin 预编译第三方库
5. IgnorePlugin：忽略不需要的模块

---

## 10. 如何优化打包体积？

1. Tree Shaking：移除未使用代码
2. Code Splitting：splitChunks + 动态 import
3. 压缩：TerserPlugin + CssMinimizerPlugin
4. Scope Hoisting：合并模块减少闭包
5. externals：CDN 资源排除出打包
6. 按需引入：`babel-plugin-import`
