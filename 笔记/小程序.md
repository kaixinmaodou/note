# 微信小程序

## 基础概念

### 小程序与 Web 的区别

| 对比 | Web | 微信小程序 |
| --- | --- | --- |
| 运行环境 | 浏览器 | **微信客户端**（JSCore + 渲染层） |
| 渲染方式 | 单线程 | **双线程**（逻辑层 + 渲染层） |
| DOM 操作 | 支持 | **不支持**（无 document/window） |
| 样式 | CSS | **WXSS**（rpx 单位） |
| 模板 | HTML | **WXML**（`wx:if`、`wx:for`） |
| 网络请求 | XHR / Fetch | **wx.request()** |
| 包大小限制 | 无 | 主包 2MB，分包总计 20MB |

### 双线程架构

```
渲染层（Webview）        逻辑层（JSCore）
  ↕                        ↕
    ← Native 作为中间桥梁 →
```

- **渲染层**：每个页面一个 Webview 线程，负责 WXML + WXSS 渲染
- **逻辑层**：所有页面共用一个 JSCore 线程，执行 JS 逻辑
- **通过 Native 层通信**：setData 数据从逻辑层序列化后传到渲染层

**性能影响：** `setData` 是跨线程通信，数据量过大或频率过高会导致卡顿。

### 项目结构

```
├── app.js          # 小程序逻辑
├── app.json        # 全局配置（页面路由、窗口样式）
├── app.wxss        # 全局样式
├── pages/

```

## 数据绑定与事件

### 数据绑定

```html
<!-- WXML -->
<view>{{ message }}</view>
<view>{{ list.length > 0 ? '有数据' : '无数据' }}</view>
```

```js
// JS
Page({
  data: {
    message: 'Hello',
    list: []
  }
});
```

### 修改数据

```js
// ✅ 正确: 通过 setData 修改
this.setData({ message: '新值' });

// 可以修改嵌套路径
this.setData({ 'obj.name': '张三', 'list[0]': 'new item' });

// ❌ 错误: 直接赋值不会触发视图更新
this.data.message = '新值';
```

> **`setData` 是同步的**，数据会立刻更新到 data 中，但视图更新是异步的。

### 事件系统

```html
<!-- bind: 冒泡事件 -->
<button bindtap="handleTap">点击</button>

<!-- catch: 阻止冒泡 -->
<view catchtap="handleOuter">
  <button bindtap="handleInner">点击</button>
</view>

<!-- 传参: 通过 data-xxx -->
<button bindtap="handleTap" data-id="{{ item.id }}">点击</button>
```

```js
handleTap(e) {
  const id = e.currentTarget.dataset.id;
  console.log(id);
}
```

### 列表渲染

```html
<view wx:for="{{ list }}" wx:key="id">
  <text>{{ index }}: {{ item.name }}</text>
</view>

<!-- 自定义变量名 -->
<view wx:for="{{ list }}" wx:for-index="i" wx:for-item="person" wx:key="id">
  <text>{{ i }}: {{ person.name }}</text>
</view>
```

### 条件渲染

```html
<!-- wx:if: 条件为 false 时不渲染（惰性） -->
<view wx:if="{{ score >= 90 }}">优秀</view>
<view wx:elif="{{ score >= 60 }}">及格</view>
<view wx:else>不及格</view>

<!-- hidden: 始终渲染，通过 display:none 切换 -->
<view hidden="{{ !show }}">内容</view>
```

## 生命周期

### App 生命周期

```js
App({
  onLaunch() {},  // 小程序初始化（全局只触发一次）
  onShow() {},    // 小程序切前台
  onHide() {}     // 小程序切后台
});
```

### Page 生命周期

```js
Page({
  onLoad(options) {},   // 页面加载（可获取路由参数）
  onShow() {},          // 页面显示
  onReady() {},         // 页面初次渲染完成
  onHide() {},          // 页面隐藏
  onUnload() {}         // 页面卸载
});
```

### Component 生命周期

```js
Component({
  lifetimes: {
    created() {},       // 组件创建
    attached() {},      // 组件进入页面节点树
    detached() {}       // 组件离开页面节点树
  }
});
```

## 页面路由与导航

```js
// 保留当前页面，跳转到新页面（栈中最多 10 层）
wx.navigateTo({ url: '/pages/detail/detail?id=1' });

// 关闭当前页面，跳转
wx.redirectTo({ url: '/pages/index/index' });

// 关闭所有页面，跳转
wx.reLaunch({ url: '/pages/index/index' });

// 跳转到 tabBar 页面
wx.switchTab({ url: '/pages/home/home' });

// 返回上一页
wx.navigateBack({ delta: 1 });
```

## 网络请求

```js
wx.request({
  url: 'https://api.example.com/data',
  method: 'POST',
  header: { 'Content-Type': 'application/json' },
  data: { page: 1 },
  success(res) {
    console.log(res.data);
  },
  fail(err) {
    console.error(err);
  }
});
```

**注意：**
- 必须使用 **HTTPS** 协议
- 需要在小程序后台配置**合法域名**
- 并发上限 **10 个**
- 开发阶段可勾选「不校验合法域名」

## 本地存储

```js
// 同步
wx.setStorageSync('key', { name: '张三' });
const data = wx.getStorageSync('key');
wx.removeStorageSync('key');

// 异步
wx.setStorage({ key: 'key', data: { name: '张三' } });
wx.getStorage({ key: 'key', success(res) { console.log(res.data); } });
```

**限制：** 单个 key 最大 1MB，总计 10MB，永久存储。

## 分包加载

```json
// app.json
{
  "pages": ["pages/index/index"],
  "subpackages": [
    {
      "root": "packageA",
      "pages": ["pages/detail/detail"]
    }
  ],
  "preloadRule": {
    "pages/index/index": {
      "network": "all",
      "packages": ["packageA"]
    }
  }
}
```

**限制：** 主包 ≤ 2MB，总计 ≤ 20MB

## 性能优化

1. **减少 setData 数据量**：只传变化的数据，避免传整个大对象
2. **合并 setData**：多次修改合并为一次调用
3. **长列表优化**：虚拟列表（`recycle-view`）
4. **图片优化**：使用 CDN、lazy-load、合适尺寸
5. **分包加载**：减小主包体积
6. **避免频繁切换 wx:if**：用 `hidden` 替代

# UniApp

## 概述

UniApp 是一个使用 Vue.js 开发跨平台应用的框架，一套代码可编译到：

| 平台 | 说明 |
| --- | --- |
| H5 | Web 端 |
| 微信/支付宝/百度/... 小程序 | 各平台小程序 |
| App（iOS / Android） | 原生 App（基于 Weex/nvue） |
| 快应用 | 快应用平台 |

### 技术栈

- **语法**：Vue 2 / Vue 3（推荐 Vue 3 + `<script setup>`）
- **样式**：支持 CSS / Less / Scss，使用 **rpx** 作为尺寸单位
- **路由**：`pages.json` 配置式路由（类似小程序）
- **API**：`uni.xxx()` 统一 API（兼容各平台）
- **状态管理**：Vuex / Pinia
- **UI 框架**：uni-ui / uView / ThorUI 等

### 与原生小程序的对比

| 对比 | 原生小程序 | UniApp |
| --- | --- | --- |
| 语法 | WXML + WXSS + JS | **Vue SFC**（.vue 文件） |
| 跨平台 | 仅微信 | **多端** |
| 生态 | 微信生态 | Vue 生态 + 插件市场 |
| 学习成本 | 专有语法 | **Vue 即可** |
| 性能 | 原生最优 | 接近原生 |

## 生命周期

UniApp 同时支持 **Vue 生命周期** 和 **小程序页面生命周期**。

### 页面生命周期（小程序风格）

```js
export default {
  onLoad(options) {},   // 页面加载
  onShow() {},          // 页面显示
  onReady() {},         // 初次渲染完成
  onHide() {},          // 页面隐藏
  onUnload() {},        // 页面卸载
  onPullDownRefresh() {},  // 下拉刷新
  onReachBottom() {},      // 触底加载
  onShareAppMessage() {}   // 分享
};
```

### Vue 3 Composition API

```vue
<script setup>
import { onMounted, ref } from 'vue';
import { onLoad, onShow, onPullDownRefresh } from '@dcloudio/uni-app';

const list = ref([]);

onLoad((options) => {
  console.log(options);
});

onShow(() => {
  loadData();
});

onPullDownRefresh(() => {
  loadData().then(() => uni.stopPullDownRefresh());
});

async function loadData() {
  const res = await uni.request({ url: 'https://api.example.com/list' });
  list.value = res.data;
}
</script>
```

## 条件编译

UniApp 的核心能力——针对不同平台编写差异化代码。

```html
<!-- #ifdef MP-WEIXIN -->
<view>仅微信小程序显示</view>
<!-- #endif -->

<!-- #ifdef H5 -->
<view>仅 H5 显示</view>
<!-- #endif -->

<!-- #ifndef APP-PLUS -->
<view>除 App 外都显示</view>
<!-- #endif -->
```

```js
// #ifdef MP-WEIXIN
wx.login({ /* ... */ });
// #endif

// #ifdef H5
window.location.href = '/login';
// #endif
```

**常用平台标识：** `H5`、`MP-WEIXIN`、`MP-ALIPAY`、`APP-PLUS`

## 自定义组件

```vue
<!-- components/my-card.vue -->
<template>
  <view class="card" @tap="handleTap">
    <text>{{ title }}</text>
    <slot />
  </view>
</template>

<script setup>
defineProps({ title: { type: String, default: '' } });
const emit = defineEmits(['click']);
const handleTap = () => emit('click');
</script>

<style scoped>
.card { padding: 20rpx; background: #fff; border-radius: 12rpx; }
</style>
```

> UniApp 支持 **easycom**，`components` 目录下的组件自动注册，无需手动 import。
